Installation Steps

  1. Database Setup
  2. Upload Files
  3. Testing the Script
  4. Cron Job Setup
  5. Customization

A) Database Setup - Back to top

To start, create a new database using phpMyAdmin or use an existing blank one. Open the file `database.sql` and replace `http://www.yourdomain.com/` and `admin@yourdomain.com` with your own domain and email details. Import the updated `database.sql` into your database via phpMyAdmin.



Next: Open the `config.php` file located in the `/includes` directory and edit the database credentials to match your setup.

B) Upload Files - Back to top

Upload all files from the 'Upload' folder to your chosen directory (preferably the root directory, such as `/public_html/`). You can use an FTP client like FileZilla for ease of use—simply enter your FTP credentials and upload the files.

C) Testing the Script - Back to top

Once all files are uploaded, visit the following URL to access the game:
http://www.yourdomain.com/gamefolder (Replace `yourdomain.com/gamefolder` with the actual path where you uploaded the script.)

Click the 'Login' button, and use the following credentials:

Username: admin
Password: 123456

Access the game’s Admin panel by clicking 'Admin' in the Game Menu to adjust settings.

Note: It’s recommended to change the default login credentials to secure your account.

D) Cron Job Setup - Back to top

To automate certain game functions, you’ll need to set up Cron Jobs. These are typically supported by most hosting providers and can be configured through cPanel. If your host does not support Cron Jobs, you may want to explore other options or manually run the scripts.

E) Customization - Back to top

1. To change the header or other images, access the `images` folder.

2. You can edit the logo PSD file located in the 'Extras' folder.

3. To customize the appearance of the game, modify the `includes/style.css` file.

4. For layout changes, you can edit `includes/header.php` and `includes/footer.php` (recommended only for experienced developers).

5. Gameplay scripts are located in the root folder and should only be altered by experienced users.
